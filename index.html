Resistor sorter

Sorting resistors is no longer futile

Our team

About us

Kilograms of resistors are thrown away every year by the EEE department at Imperial

This waste is caused by hurried students always going to the drawers of sorted resistors so they can be sure of picking up a resistor of the correct value immediately without having to search and check the colour codes of each resistor. Many of these resistors get used once on a breadboard prototype and are simply put in to a box at the end of the session with the hope that they'll be used again.

However, this is very rarely the case because it is much more difficult to search through a large box for one specific resitor value. They remain unsorted because it is simply more cost effective to buy more than it is to pay someone for the time to sort them. It was this wastage that inspired a product that would sort all the resistors automatically. A device that sorted resitors automatically would mean places of electronic prototyping (such as hackspaces and universities!) would save lots of money and produce much less electrical waste.

Our Product

As solution was developed to stop this waste

In the few months we had to deliver this product, and after multiple design iterations of the subsections, we finished up with a compact and functional device. The device is a proof on concept project so it has been assembled with quick design adjustments in mind, rather than spending our time on developing the aesthetics required for a marketable product.

To use the machine, simply place the resistors on the conveyor belt, spread out enough so the conveyor belt is able to drop them in to the correct compartment individually. Press the start button to begin and the resistors will then travel along the belt. Once the computer vision system has determined its value from the coloured stripes, they will be dropped in to the correct compartment. The conveyor belt will stop after having run for 3 seconds without any resistors detected. If desired, you can simply store your sorted resistors in the rotary compartments but if you already have a place to store your ressitors, the individual compartments are easily removable for convenience.

Q: How do I register and set up my own custom doamin?

  1. Check if the domain you want is free - This can be done from many websites such as domaincheck.
  2. Add to GitHub Pages - Go to the repository settings and simply paste your domain in to the custom domain box and then press save.
  3. Buy the domain - There are many DNS providers where you can buy the domain such as domaincheck. Be sure to check if the DNS provider supports ALIAS or ANAME records.
  4. Set up your custom domain - Set up the ALIAS or ANAME record with the specific instructions from your chosen DNS provider
  5. Enable HTTPS - Remove then re-add your custom doamin to the GitHub Pages settings and GitHub will automatically enable HTTPS

    All sections must have clear titles and be easy to find to ensure the the user can find the information required
  • FLEXIBLE VISUALS

  • CONNECTIVITY

  • BREATHING SPACE

  • TYPOGRAPHY

Q: What accessibility aspects do I need to consider?

There are many aspects to consider to make a website accessible to as many people as possible. This includes such guidelines as not relying on colour in case the user is colour blind and using "alt tags" on pictures so a screen reader can describe the image to the user. Also any videos present on the website should be subtitiled. To cater for those with mobility issues, ensure all buttons are not too small to click and in the worst case, the website should be usable with the keyboard alone. The webstie should also work across all common devices. It may aslo be useful to include an accessibilty guide explaining methods of solving various problems such as the excellent example on the BBC webstie. It is a legal requirement that businesses with an online presence must provide accessibilty to disabled users but there is the Web Content Accessibility Guidelines (WCAG) to help achieve this

Q: What is GDPR and what do I need to do about it?

The General DataProtection Regulation (GDPR) is an EU regulation focusing on data protection and privacy. This website doesn't collect any data whatsoever but if it did, the following guidelines would need to be followed

  • Consent
    Personal data may not be processed without the consent of the user apart from a few circumstances such as to protect the vital interests of the user. THe user can revoke this consent at any time
  • Transparency
    The user must be informed about what data of theirs is being collected and what it'll be used for and how long it will be kept. They must also be informed of their privacy rights
  • Right of access
    The user must be able to view their personal data
  • Records of processing
    Records of all the processing done on data must be kept and available to the supervising authority on request
  • Security
    Where possible, the data must be anonymised (ensure it is impossible to identify the person from the data) or undergo pseudonymisation. This is where the data is put through an irreversible process to make the data unusable without the use of a key. An example is encryption.
  • Action for data breaches
    If personal data does get leaked, despite the step above, then you must legally notify the supervisory authority and the users if the breach will have an adverse affect on them
  • Right to erasure
    The user has the right to request that all their personal data be deleted within 30 days
  • Responsibility
    It is the responsibilty of the data controller to demonstrate compliance to the GDPR rules
